# Супер Аналитик (Gerald Elite Trader) - Концепция и Роадмап

## Цель

Создать когнитивный модуль (скилл) для системы Gerald, который будет детально анализировать входящие сигналы от `Gerald-Sniper`, оценивать вероятности, контекст старших таймфреймов (Smart Money / ICT) и формировать полноценные торговые планы (Playbooks) с несколькими сценариями развития событий.

## Концепции для реализации (Архитектура Когнитивных Способностей)

### 1. Multi-Agent Debate (Спор Агентов)

Вместо одного LLM-ответа используются три роли:

- **Бык (Профит-Хантер):** Ищет причины, почему сделка идеальна (потенциал роста, лонговые аргументы).
- **Медведь (Риск-Менеджер):** Жестко критикует сетап, ищет скрытые пулы ликвидности для снятия стопов, оценивает вероятность ложного пробоя.
- **Арбитр (Elite Trader):** Изучает аргументы Быка и Медведя, выносит финальный вердикт (TAKE/SKIP) и строит план защиты.

### 2. ICT / Smart Money RAG-Оракул

- Аналитик оперирует не просто классическим ТА, а зонами ликвидности (Order Blocks, FVG/Imbalance, Liquidity Pools) на таймфреймах H1/H4.
- Способность "схлопывать" сигнал со старшим таймфреймом: например, отмена входа по рынку в пользу лимитного ордера ниже текущей цены для сбора ликвидности перед полетом.

### 3. Сценарное Дерево (Tree of Thoughts) + Trade Playbook

Генерация Торгового Плана (Trade Playbook), который содержит сценарии:

- **Сценарий А (Позитивный):** Цели для фиксации, точки переноса стопа в безубыток.
- **Сценарий Б (Стоп-ран):** План действий, если цену повели за ликвидностью ниже уровня входа.
- **Сценарий В (Макро-шок):** Действия в случае резкого отката BTC.

### 4. Tool-Using Quant (Гибрид Математики и ИИ)

Предоставление агенту доступа к функциям-инструментам (Tools) перед выдачей ответа:

- Запрос Volume Profile (профиля объема)
- Запрос агрегированного открытого интереса (OI)
- Проверка каскада ликвидаций на фьючерсах

## Следующие шаги (Для новой сессии)

1. Создать новую директорию скилла `skills/gerald-elite-trader`.
2. Настроить `SmartRouter` на использование сложных промптов и вызов агентов по очереди (Debate).
3. Интегрировать скилл с шиной сообщений (или слушать БД `alerts`), чтобы он "просыпался" при новом сигнале от Снайпера.
4. Разработать системный промпт (SYS_PROMPT) для каждой роли (Арбитр, Медведь, Бык).
5. Настроить мост между Telegram-ответом Analyst и Trade Tracker (передача сложных параметров).
