# ТЕХНИЧЕСКОЕ ЗАДАНИЕ (ТЗ) v2.0: "Gerald-SuperBrain"

**Статус:** Утверждено к разработке (Strict Constraints Edition)
**Дата:** 2026-02-23

---

## 1. АРХИТЕКТУРНЫЙ СТЕК (Strictly Enforced)

**ЗАПРЕЩАЕТСЯ** использовать тяжеловесные AI-фреймворки (LangChain, LlamaIndex, AutoGen). Вся логика агента пишется на чистом **Python + asyncio**.

- **Инференс LLM:** `llama-cpp-python` (Сервер + Python API).
  - _Почему:_ Полный контроль над VRAM (`n_gpu_layers`) и аппаратная поддержка Grammar/JSON Schema.
- **Vector DB (Память):** `LanceDB` или `SQLite-vec`.
  - _Почему:_ Библиотеки, а не серверы. Живут в одном файле `memory.lance`, используют 0% ресурсов в простое.
- **Embeddings:** `nomic-embed-text-v1.5` (через llama.cpp) или `MiniLM-L12-v2`.

---

## 2. ЯДРО АГЕНТА: СТРУКТУРИРОВАННЫЙ ВЫВОД

Локальные модели 7B обязаны работать через **Constrained Decoding**.

1.  Все инструменты (Tools) описываются через **Pydantic-схемы**.
2.  Вызов `llama-cpp-python` содержит параметр `response_format` (или grammar), сгенерированный из Pydantic.
3.  Гарантия 100% валидного JSON.

**Стандарт вызова:**

```json
{
  "thought": "Мысли агента на русском",
  "tool_name": "read_file",
  "tool_args": {}
}
```

---

## 3. МОДУЛЬ ИНДЕКСАЦИИ (Background RAG)

1.  **BackgroundIndexer:** Асинхронный воркер для мониторинга `gerald_workspace` и документов.
2.  **Приоритет ОС:** Использование `psutil` для установки пониженного приоритета процесса индексации.
3.  **Semantic Splitter:** Умная разбивка на чанки.
4.  **Reranker:** `bge-reranker-v2-m3` в GGUF формате (используется только при ответе).

---

## 4. ИЗОЛЯЦИЯ КОДА (Sandbox)

1.  Инструмент `execute_python_code` сохраняет код в `gerald_workspace/tmp_run/`.
2.  Запуск через `subprocess.run` с:
    - Ограничением сети (изоляция).
    - Таймаутом: **15 секунд**.
    - Обратной связью об ошибке таймаута.

---

## 5. СТАНДАРТЫ РАЗРАБОТКИ (Guidelines)

1.  **Типизация:** 100% покрытие `type hints`. Проверка через `mypy`.
2.  **Логирование:** Только `loguru`. Ротация логов 10 МБ. Файл: `gerald_sys.log`.
3.  **Graceful Shutdown:** Обработка `Ctrl+C`, сохранение сессии в SQLite, выгрузка VRAM.
4.  **Конфигурация:** Все настройки в `config.yaml`. Никакого хардкода.

---

## 6. КРИТЕРИИ ПРИЕМКИ (Тесты)

1.  **Тест "Глупый парсер":** Валидация JSON при сложных схемах.
2.  **Тест "Тихий ход":** Фоновая индексация 50+ файлов без фризов чата.
3.  **Тест "Взбунтовавшийся скрипт":** Убийство бесконечного цикла через 15с.
4.  **Проверка VRAM:** Использование GPU < 7.8 GB (генерация + RAG).
